## <a name="enable-in-process-hosting"></a><span data-ttu-id="6151f-108">启用进程内托管</span><span class="sxs-lookup"><span data-stu-id="6151f-108">Enable in-process hosting</span></span>

## <a name="general-architecture"></a><span data-ttu-id="6151f-111">一般体系结构</span><span class="sxs-lookup"><span data-stu-id="6151f-111">General architecture</span></span>

<span data-ttu-id="6151f-112">请求的常规流程如下：</span><span class="sxs-lookup"><span data-stu-id="6151f-112">The general flow of a request is as follows:</span></span>

1. <span data-ttu-id="6151f-113">请求从 Web 到达内核模式 HTTP.sys 驱动程序。</span><span class="sxs-lookup"><span data-stu-id="6151f-113">A request arrives from the web to the kernel-mode HTTP.sys driver.</span></span>
1. <span data-ttu-id="6151f-114">驱动程序将本机请求路由到网站的配置端口上的 IIS，通常为 80 (HTTP) 或 443 (HTTPS)。</span><span class="sxs-lookup"><span data-stu-id="6151f-114">The driver routes the native request to IIS on the website's configured port, usually 80 (HTTP) or 443 (HTTPS).</span></span>
1. <span data-ttu-id="6151f-115">ASP.NET Core 模块接收本机请求，并将其传递给 IIS HTTP 服务器 (`IISHttpServer`)。</span><span class="sxs-lookup"><span data-stu-id="6151f-115">The ASP.NET Core Module receives the native request and passes it to IIS HTTP Server (`IISHttpServer`).</span></span> <span data-ttu-id="6151f-116">IIS HTTP 服务器是将请求从本机转换为托管的 IIS 进程内服务器实现。</span><span class="sxs-lookup"><span data-stu-id="6151f-116">IIS HTTP Server is an in-process server implementation for IIS that converts the request from native to managed.</span></span>

<span data-ttu-id="6151f-117">在 IIS HTTP 服务器处理请求后：</span><span class="sxs-lookup"><span data-stu-id="6151f-117">After the IIS HTTP Server processes the request:</span></span>

1. <span data-ttu-id="6151f-118">请求被发送到 ASP.NET Core 中间件管道。</span><span class="sxs-lookup"><span data-stu-id="6151f-118">The request is sent to the ASP.NET Core middleware pipeline.</span></span>
1. <span data-ttu-id="6151f-119">中间件管道处理该请求并将其作为 `HttpContext` 实例传递给应用的逻辑。</span><span class="sxs-lookup"><span data-stu-id="6151f-119">The middleware pipeline handles the request and passes it on as an `HttpContext` instance to the app's logic.</span></span>
1. <span data-ttu-id="6151f-120">应用的响应通过 IIS HTTP 服务器传递回 IIS。</span><span class="sxs-lookup"><span data-stu-id="6151f-120">The app's response is passed back to IIS through IIS HTTP Server.</span></span>
1. <span data-ttu-id="6151f-121">IIS 将响应发送到发起请求的客户端。</span><span class="sxs-lookup"><span data-stu-id="6151f-121">IIS sends the response to the client that initiated the request.</span></span>

<span data-ttu-id="6151f-122">`CreateDefaultBuilder` 添加 <xref:Microsoft.AspNetCore.Hosting.Server.IServer> 实例的方式是：调用 <xref:Microsoft.AspNetCore.Hosting.WebHostBuilderIISExtensions.UseIIS%2A> 方法来启动 [CoreCLR](/dotnet/standard/glossary#coreclr) 和将应用托管在 IIS 工作进程（`w3wp.exe` 或 `iisexpress.exe`）内。</span><span class="sxs-lookup"><span data-stu-id="6151f-122">`CreateDefaultBuilder` adds an <xref:Microsoft.AspNetCore.Hosting.Server.IServer> instance by calling the <xref:Microsoft.AspNetCore.Hosting.WebHostBuilderIISExtensions.UseIIS%2A> method to boot the [CoreCLR](/dotnet/standard/glossary#coreclr) and host the app inside of the IIS worker process (`w3wp.exe` or `iisexpress.exe`).</span></span> <span data-ttu-id="6151f-123">性能测试表明，与在进程外托管应用并将请求代理传入 [Kestrel](xref:fundamentals/servers/kestrel) 相比，在进程中托管 .NET Core 应用可提供明显更高的请求吞吐量。</span><span class="sxs-lookup"><span data-stu-id="6151f-123">Performance tests indicate that hosting a .NET Core app in-process delivers significantly higher request throughput compared to hosting the app out-of-process and proxying requests to [Kestrel](xref:fundamentals/servers/kestrel).</span></span>

<span data-ttu-id="6151f-124">作为单个文件可执行文件发布的应用无法由进程内托管模型加载。</span><span class="sxs-lookup"><span data-stu-id="6151f-124">Apps published as a single file executable can't be loaded by the in-process hosting model.</span></span>

## <a name="differences-between-in-process-and-out-of-process-hosting"></a><span data-ttu-id="6151f-142">进程内和进程外托管之间的差异</span><span class="sxs-lookup"><span data-stu-id="6151f-142">Differences between in-process and out-of-process hosting</span></span>

<span data-ttu-id="6151f-143">在进程内托管时，将应用以下特征：</span><span class="sxs-lookup"><span data-stu-id="6151f-143">The following characteristics apply when hosting in-process:</span></span>

  * <span data-ttu-id="6151f-146">注册 `IISHttpServer`。</span><span class="sxs-lookup"><span data-stu-id="6151f-146">Register the `IISHttpServer`.</span></span>

  * <span data-ttu-id="6151f-148">配置主机以捕获启动错误。</span><span class="sxs-lookup"><span data-stu-id="6151f-148">Configure the host to capture startup errors.</span></span>
